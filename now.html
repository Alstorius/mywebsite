Now

dernière mise à jour : Avril 2026

// travail

Je travaille comme architecte systèmes sur une plateforme IoT industrielle. La majorité de mon temps tourne autour de la conception de pipelines de données embarqués et de la réduction de la dette technique d'un système qui a grandi plus vite que son design.

Actuellement en train de migrer des composants critiques vers Rust. Pas pour suivre une mode — parce que les garanties de mémoire changent vraiment la nature des conversations qu'on a sur les bugs en production.

// projets

  • Un petit outil CLI en Rust pour automatiser des tâches de release — rien de révolutionnaire, mais ça m'apprend.
  • Ce site — construire quelque chose de propre, sans framework, me repose l'esprit.
  • Un essai en cours sur la complexité cognitive dans les systèmes distribués.

// lecture

  • A Philosophy of Software Design — Ousterhout. À la moitié. Dense, juste, inconfortable.
  • The Mythical Man-Month — Relécteur. Toujours aussi déprimant d'actualité.

// dans ma tête

Je pense beaucoup à la différence entre comprendre un système et être capable de le décrire. On confond souvent les deux. La documentation n'est pas la connaissance.

Je me demande aussi si le mouvement vers l'IA dans le code ne va pas, paradoxalement, aggraver la dette de compréhension des systèmes. On va écrire plus vite du code que personne ne comprend vraiment.

C'est une page /now — une convention initiée par Derek Sivers. L'idée : écrire ce sur quoi vous travaillez en ce moment, comme si un ami vous demandait ce qui se passe dans votre vie.
